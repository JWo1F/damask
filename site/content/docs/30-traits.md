## `Component`

```rust
pub trait Component: Render {
    fn default_renderer(&self) -> Box<dyn Renderer>;
    fn render(&self) -> String;
    fn render_with(&self, slots: Slots<'_>) -> String;
}
```

Generated by the derive. `render` is `render_with(Slots::EMPTY)`, and both build
a renderer chosen by the template's host language. Object-safe, so `&dyn Component`
and `Vec<Box<dyn Component>>` work.

## `Render`

```rust
pub trait Render {
    fn render_into(&self, r: &mut dyn Renderer);
    fn render_slots(&self, r: &mut dyn Renderer, slots: Slots<'_>) { тАж }
}
```

The lower-level half: write yourself into a renderer someone else owns. The
derive implements `render_slots` with the lowered template and points
`render_into` at it with `Slots::EMPTY`.

The default `render_slots` ignores its slots, which suits content that has no
slots of its own тАФ a `Fragment`, or a hand-written impl тАФ and lets such an impl
stay a single method.

`Render` is implemented for `Box<T>` where `T: Render + ?Sized`, so
`Box<dyn Render>` works for heterogeneous children.

## `Renderer`

The output buffer and the escaping policy, in one object-safe trait.

| Method | Backs | Default |
|---|---|---|
| `write_raw(&str)` | tags and safe content | required |
| `write_escaped(&dyn Display)` | `{ тАж }` | required |
| `finish(self: Box<Self>) -> String` | producing the output | required |
| `write_text(&str)` | literal text between tags | forwards to `write_raw` |
| `write_display_raw(&dyn Display)` | `{@html тАж }` | formats through `write_raw` |
| `push_indent` / `pop_indent` | layout | no-op |
| `close_line(depth)` | layout | no-op |
| `set_verbatim(bool)` | `<pre>`, `<script>`, `<style>`, `<textarea>` | no-op |

Everything with a default exists so that a renderer written before those hooks
did stays correct without being touched тАФ and so a minimal renderer is three
methods.

`write_text` is separate from `write_raw` because it is the only markup a
renderer may lay out: it is the whitespace *between* elements, which HTML
collapses anyway. A tag's own bytes are never re-laid-out, since the only newline
one can contain is inside an attribute value, and re-indenting a multi-line
`title` would change what it says.

## Why a child renders through its parent's renderer

`{@render}` and component elements write into the **same** renderer as their
parent. Escaping is therefore a property of the output, not of the component: a
component cannot escape differently depending on who called it.
